# Add provenance comments to generated delegate commands

## Description

The `NuruDelegateCommandGenerator` generates Command/Query classes, but the output doesn't include any information about what source code triggered the generation. This makes debugging difficult when unexpected classes are generated (like `Default_Generated_Query`).

Add provenance information to generated code so developers can trace back to the source.

## Checklist

- [ ] Add provenance fields to `DelegateCommandInfo` record in `nuru-delegate-command-generator.types.cs`
  - `SourceFilePath` - path to the source file
  - `SourceLineNumber` - line number of the AsCommand/AsQuery call
  - `PatternText` - the original route pattern string
  - `HandlerType` - description of handler (lambda, method group, etc.)
  - `HandlerSkipReason` - if handler wasn't generated, explain why
- [ ] Extract source location in `ExtractCommandInfo` using `context.Node.GetLocation()`
- [ ] Update `GenerateCommandClass` in `nuru-delegate-command-generator.codegen.cs` to emit provenance as XML `<remarks>`
- [ ] Test with samples to verify provenance appears in generated output

### Why This Helps

With provenance, when you see `Default_Generated_Query`, you immediately know:
1. **What file** triggered it
2. **What pattern** was parsed (reveals naming issue with `--capabilities`)
3. **Why no handler** was generated (method group not supported yet)

This is prerequisite for fixing task 232 (sample compilation warnings) since we need to understand what's being generated before fixing it.
